## 2단계: 논문 리뷰 신청 — 처리 방법

### 신청서에 이렇게 써있을 거예요

```
이름: 김대학
이메일: kim@univ.ac.kr
서비스: 논문 리뷰
논문 제목: 딥러닝 기반 이미지 분류에 관한 연구
파일: paper.pdf
```

### 처리 순서 (총 15분)

**① Claude Code 실행**

바탕화면에서 `claude` 입력 → 엔터

**② 아래 명령어 통째로 복사해서 붙여넣기**

```
@narratologist @psychologist @anthropologist @model-qa-specialist @language-translator

아래 논문을 6가지 영역에서 분석해주세요.

각 영역마다 "문제점"과 "개선 제안"을 구체적으로 써주세요.
한국어로 답변해주세요.

[분석 영역]
1. 논리 구조: 연구 질문→가설→방법론→결론 흐름이 논리적인가?
2. 가독성: 초록만 읽고 핵심 파악 가능한가? 문장이 너무 길지 않은가?
3. 편향: 표본이 특정 집단에 치우치지 않았나? 결론 일반화가 과도한가?
4. 방법론: 통계 검정 선택 적절한가? p-hacking 의심되는가? 결측치 처리는?
5. 영문 초록: 한글-영문 일치하는가? 학술 표현은 적절한가?
6. 종합 평가: Critical(필수보완) / Recommended(권장보완) / 투고 가능성 예측

---
[논문 내용 시작]
(여기에 논문 PDF 내용을 복사해서 붙여넣으세요)
[논문 내용 끝]
```

**③ 논문 PDF 열기 → 전체 선택(Ctrl+A) → 복사(Ctrl+C) → 붙여넣기**

`(여기에 논문 PDF 내용을 복사해서 붙여넣으세요)` 이 부분에 붙여넣으세요.

**④ 엔터 치고 기다리기 (보통 3~5분)**

AI가 분석하는 동안 커피 한 잔 하세요.

**⑤ 결과가 나오면?**

```
@document-generator
방금 나온 논문 리뷰 결과를 깔끔한 PDF 리포트 형식으로 정리해줘.

포맷:
- 표지: 논문 제목, 리뷰일자
- 영역 1: 논리 구조 분석
- 영역 2: 가독성 평가
- 영역 3: 편향 검증
- 영역 4: 방법론 검증
- 영역 5: 영문 초록 검토
- 영역 6: 종합 평가 (Critical / Recommended)

전문적인 톤으로.
```

**⑥ 결과 복사해서 구글 문서에 붙여넣기**

- docs.google.com → 새 문서
- AI 결과 전체 복사해서 붙여넣기
- 파일 → 다운로드 → PDF 문서(.pdf)
- 파일 이름: `[논문제목]_리뷰리포트_20260615.pdf`

**⑦ 이메일 보내기**

```
제목: [AI 논문 리뷰] OOO님의 논문 리뷰 결과입니다

안녕하세요 OOO님,

요청하신 논문 리뷰 결과를 보내드립니다.

📎 첨부파일을 확인해주세요.

리뷰는 6가지 영역(논리구조·가독성·편향·방법론·영문초록·종합평가)으로 진행되었습니다.

⚠️ 중요: 이 리뷰는 AI의 1차 분석입니다.
투고 전 반드시 지도교수님의 확인을 받으세요.

감사합니다.
AI 연구소 드림
```

**⑧ 구글폼 응답 시트에 "처리완료" 표시**

### 처리 순서

**① 신청자에게 확인 메일 보내기**

```

🎯 업종: SaaS
📊 난이도: 중급

세션 당일, 아래 링크로 접속해주세요:
👉 [Google Meet 링크] (또는 "카카오톡 오픈채팅으로 진행됩니다" 라고 써도 됨)

준비물:
- 본인 회사/제품에 대한 1분 소개 (있으면 좋음)
- 연습하고 싶은 특정 상황이 있다면 미리 알려주세요

감사합니다!
```

**② 당일 세션 준비**

Claude Code에 아래 명령어 붙여넣기:

```
@discovery-coach @deal-strategist @behavioral-nudge-engine @strategy-duel-agent

지금부터 B2B 영업 시뮬레이션을 진행합니다.

당신은 가상 바이어입니다.

[바이어 설정]
- 업종: SaaS (마케팅 자동화 툴 구매 검토 중)
- 직급: CMO (마케팅 총괄 임원)
- 회사 규모: 300명, 연매출 500억
- 현재 상황: Salesforce Marketing Cloud 사용 중, 연 5천만원 계약
- 숨은 니즈: 팀 생산성 저하, 데이터 연동 문제로 매주 10시간씩 수작업
- 성격: ROI에 매우 민감, 데이터 기반 의사결정, 과거 SaaS 도입 실패 경험 있음

[시뮬레이션 규칙]
- 난이도: 중급
- 바이어는 절대 먼저 솔루션을 요청하지 않는다
- SPIN의 Implication 질문이 나올 때까지 진짜 니즈를 감춘다
- 가격 관련 반론을 최소 1회 제기한다
- 10턴 정도 대화 후 "오늘 미팅은 여기까지 하시죠"로 마무리

시작: "안녕하세요, 만나서 반갑습니다. 오늘 저희에게 어떤 도움을 드릴 수 있을까요?"
```

## 5단계: 문제 생겼을 때

| 문제 | 해결 방법 |
|------|-----------|
| AI 답변이 영어로 왔다 | 프롬프트 맨 앞에 "한국어로 답변해주세요" 추가 |
| 논문이 너무 길어서 잘렸다 | "이어서 계속 분석해줘" 입력 |
| @에이전트가 인식 안 됨 | Claude Code 종료 후 다시 `claude` 실행 |
| PDF 변환이 안 됨 | 결과 복사 → 구글 문서 붙여넣기 → PDF 다운로드 |
| 신청자가 너무 많다 | 구글폼에서 "응답 수락 중지" 버튼 켜기 |
