# 🤖 Hermes CronJob 자동화 설계

## 개요

Hermes에 내장된 CronJob + Skills 기능으로 전 과정을 자동화합니다.

---

## 구조

```
구글폼 신청 → Script가 폴링 → 신규 감지 → CronJob 실행
                                                ↓
                              ┌─────────────────┴─────────────────┐
                              ↓                                   ↓
                      논문 리뷰 자동화                       영업 시뮬 자동화
                              ↓                                   ↓
                        PDF 리포트 생성                    일정 확정 메일 발송
                              ↓                                   ↓
                        이메일 자동 발송                   당일 세션 실행
```

---

## Step 1: 신청서 감시 스크립트 (watchdog)

매 30분마다 구글폼 응답 시트를 확인하고, 새로운 신청이 있으면 Hermes에게 알립니다.

`check-applications.py`:

```python
import gspread
from oauth2client.service_account import ServiceAccountCredentials
import json
import os
from datetime import datetime, timedelta

LAST_CHECK_FILE = os.path.expanduser("~/.hermes/last_check.json")

scope = ["https://spreadsheets.google.com/feeds", "https://www.googleapis.com/auth/drive"]
creds = ServiceAccountCredentials.from_json_keyfile_name(
    os.path.expanduser("~/.hermes/credentials.json"), scope
)
client = gspread.authorize(creds)
sheet = client.open("AI연구소-신청서").sheet1
records = sheet.get_all_records()

last_check = {}
if os.path.exists(LAST_CHECK_FILE):
    with open(LAST_CHECK_FILE) as f:
        last_check = json.load(f)

last_row = last_check.get("last_row", 0)
new_entries = records[last_row:]

if new_entries:
    for entry in new_entries:
        entry["row"] = last_row + 1 + new_entries.index(entry)

    with open(LAST_CHECK_FILE, "w") as f:
        json.dump({"last_row": len(records), "updated": datetime.now().isoformat()}, f)

    # 중요: Hermes에게 전달할 신규 신청 정보를 stdout으로 출력
    print(json.dumps(new_entries, ensure_ascii=False, indent=2))
else:
    # 신규 없음 → 조용히 종료 (아무 출력 없으면 CronJob이 알림 보내지 않음)
    pass
```

---

## Step 2: CronJob으로 연결

체크 스크립트가 새로운 신청을 감지하면 → Hermes CronJob이 자동 실행됩니다.

```
cron: "30m"  # 30분마다 실행
script: check-applications.py
prompt: >
  새로운 신청이 위에 JSON으로 출력되었습니다.

  각 신청 항목을 확인하고:

  만약 "서비스"가 "논문 리뷰"면:
  - 논문 내용을 읽고, 6가지 영역(논리·가독성·편향·방법론·영문초록·종합평가)으로 분석
  - 결과를 PDF 보고서로 정리
  - 신청자 이메일로 발송했다고 보고

  만약 "서비스"가 "영업 시뮬레이션"이면:
  - 신청자에게 일정 확정 메일 회신 (업종/난이도 반영)
  - 희망 시간이 있으면 그 시간에 맞춰 일정 확정

  각 신청 처리 후 결과를 간단히 보고해주세요.

  (신규 신청이 없으면 아무 출력도 없을 테니, 조용히 넘어가면 됩니다)
```

## Step 3: 실제 운영 시나리오

### 아침 9시, 신청 2건이 들어왔다

check-applications.py가 stdout으로 출력:

```json
[
  {
    "이름": "김대학",
    "이메일": "kim@univ.ac.kr",
    "서비스": "논문 리뷰",
    "논문파일": "https://drive.google.com/xxx/paper.pdf"
  },
  {
    "이름": "박세일",
    "이메일": "park@sales.co.kr",
    "서비스": "영업 시뮬레이션",
    "업종": "SaaS",
    "난이도": "중급"
  }
]
```

Hermes CronJob이 이걸 보고 자동 실행:

1. "논문 리뷰" → paper-reviewer skill 로드 → 논문 분석 → 리포트 생성 → 이메일 발송 완료 보고
2. "영업 시뮬" → sales-simulator skill 로드 → 신청자에게 메일로 일정 제안

**10분 만에 자동 완료. 사용자는 이메일만 받아봄.**

---

## Step 4: 기술적 구성 요약

| 구성 요소 | 역할 |
|-----------|------|
| `check-applications.py` | 30분마다 구글폼 확인, 신규 있으면 JSON 출력 |
| CronJob (30m) | 스크립트 stdout을 보고 자동으로 에이전트 실행 |
| Skills (paper-reviewer, sales-simulator) | CronJob이 로드할 작업 지침 |
| Hermes | CronJob 자동 실행 + 이메일 발송까지 일괄 처리 |

---

## Step 5: 전체 흐름

```
구글폼에 누군가 신청서 제출
         ↓
check-applications.py (30분 간격 cron 실행)
         ↓
새로운 신청 있음 → stdout에 JSON 출력
         ↓
Hermes가 JSON 읽고 자동 판단:
  ├── "논문 리뷰" → paper-reviewer Skill 로드
  │                  → 논문 분석 → PDF → 신청자 이메일
  │
  └── "영업 시뮬" → sales-simulator Skill 로드
                     → 일정 확정 메일 → 응답
         ↓
신청자는 이메일로 결과 받음
         ↓
Hermes가 "○건 처리 완료" 보고
```

## 🚀 오늘 당장 할 수 있는 것

1. `skills/paper-reviewer/` 생성 (이미 우리가 만든 문서 기반으로)
2. `skills/sales-simulator/` 생성
3. `check-applications.py` 작성
4. CronJob 등록: `cronjob(action="create", schedule="30m", script="check-applications.py", prompt=...)`

## ⚠️ 주의사항

- check-applications.py가 **stdout에 아무것도 출력하지 않으면** CronJob이 알림을 보내지 않습니다 (no_agent 모드 아님, 기본 모드)
- 신청 처리 결과는 Hermes가 최종 응답으로 사용자에게 전달
- CronJob의 Agent가 신청자 이메일을 읽고 처리해야 하므로 `web` + `file` 도구 필요
