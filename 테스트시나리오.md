# 🔧 자체 구동 가능한 CronJob — 구글폼 없이 테스트

Hermes의 CronJob이 PC 꺼져도 돌게 하려면 GitHub Actions가 필요.
하지만 CronJob 자체는 Hermes 안에서 30분마다 돈다.
이걸 바로 테스트하겠습니다.

## 테스트 시나리오

가상의 신청 2건을 만들고, CronJob이 auto-review-pipeline Skill을 로드해서 처리하는지 확인.

---

## Step 1: 가상 신청 시뮬레이션

CronJob의 prompt에 직접 가상 신청 데이터를 넣어서 테스트 실행.

```json
[
  {
    "이름": "김대학",
    "이메일": "test-paper@example.com",
    "서비스": "논문 리뷰",
    "논문 제목": "딥러닝 기반 이미지 분류 성능 향상에 관한 연구",
    "논문 내용": "초록: 본 연구는 CNN 기반 이미지 분류 모델의 성능을 향상시키기 위해 새로운 데이터 증강 기법을 제안한다. 실험 결과, 제안된 방법은 CIFAR-10에서 94.2%, ImageNet에서 78.6%의 정확도를 달성했다. 방법론: ResNet-50을 백본으로 사용하였으며, 학습률 0.001, 배치 사이즈 64, 200에폭 훈련. 결론: 제안된 증강 기법이 기존 방법 대비 우수한 성능을 보였다."
  },
  {
    "이름": "박세일",
    "이메일": "test-sales@example.com",
    "서비스": "영업 시뮬레이션",
    "업종": "SaaS",
    "난이도": "중급",
    "희망 시간": "2026-06-16 15:00"
  }
]
```

---

## Step 2: CronJob이 실행하는 실제 프롬프트

```
check-applications.py 출력 결과: 위 JSON.

이 신청들을 auto-review-pipeline Skill에 따라 처리하세요.

1. 김대학님 - 논문 리뷰:
   - 논리 구조, 가독성, 편향, 방법론, 영문초록, 종합평가 6영역 분석
   - 각 영역 문제점과 개선 제안
   - 결과 리포트

2. 박세일님 - 영업 시뮬레이션:
   - 업종: SaaS, 난이도: 중급, 희망: 6/16 15:00
   - 일정 확정 안내 메일 초안

처리 후 결과 요약 보고.
```

---

## Step 3: 기대 결과

- CronJob이 auto-review-pipeline Skill을 로드
- 김대학 논문 분석 → 6영역 리포트 출력
- 박세일 영업 시뮬 → 일정 확정 메일 초안 출력
- "2건 처리 완료" 보고
